# 00_INDEX — Índice de navegação

> Base de UX Research / Discovery / Design destilada em arquivos curtos e acionáveis. **Comece pela necessidade, não pelo método.** Não sabe nem qual CORE abrir? → `_METHOD_PICKER`. (mapa técnico da pasta = `ARQUITETURA.md`)

---

## CORE — abra por necessidade

| # | Tema | Resolve | Abra quando… |
|---|---|---|---|
| **01** | Métodos | Qual método de pesquisa usar | "não sei qual método pra esta dúvida" |
| **02** | Discovery | Vago → problema definido; reduzir o risco certo | "pedido vago, não sei qual é o problema" |
| **03** | Entrevistas e contexto | Entender o porquê (motivação, dor, decisão) | "preciso ouvir/observar usuários" |
| **04** | Síntese e mapas | Dado bruto → insight, jornada, oportunidade | "tenho dados soltos, e agora?" |
| **05** | Arquitetura da Informação (AI) | Organizar, nomear, validar estrutura | "não sei como organizar o menu / onde procuram" |
| **06** | Ideação e workshops | Oportunidade → solução testável (HMW, Crazy 8, sprint) | "tenho o problema, preciso gerar e escolher ideias" |
| **07** | Teste de usabilidade | Ver se a solução funciona (comportamento) | "tenho protótipo/tela, funciona?" |
| **08** | Métricas de UX | Medir, comparar, acompanhar experiência | "preciso medir se melhorou / onde travam em escala" |
| **09** | E-commerce | Pesquisa aplicada a loja (PDP, carrinho, checkout) | "é uma loja virtual / fluxo de compra" |
| **10** | Acessibilidade | Remover barreiras; WCAG aplicado, a11y | "preciso garantir que mais gente consiga usar" |
| **11** | ResearchOps | Operação: consentimento, LGPD, intake, repositório | "como recrutar/consentir/guardar/escalar pesquisa" |
| **12** | Produto com IA | UX de sistemas probabilísticos (confiança, fallback) | "a feature usa IA / chatbot / recomendação" |
| **14** | UX sem pesquisa formal | Evidência indireta quando pesquisa é vetada | "não posso pesquisar / só output / DS pronto" |
| **15** | Documentação | O ato de registrar (notas, insight, log, readout) | "fiz a pesquisa, como registro pra não perder?" |

*(Não existe 13. Salto 12→14→15 é proposital — ver `ARQUITETURA §2`.)*

---

## Réguas — sempre à mão

- **`_METHOD_PICKER`** — "tenho esta dúvida, qual método uso?" em uma tela + matriz risco→método. **Porta de entrada quando não sabe nem qual CORE abrir.**
- **`_GLOSSARY`** — definição de um termo; resolve a colisão de sigla IA (Inteligência Artificial) vs AI (Arquitetura da Informação).
- **`_SOURCES`** — a fonte por trás de um tema; consultar ao aprofundar ou citar.

---

## Apoio

- **`templates/`** — formulários copiáveis (intake, plano, screener, consentimento, relatório, log de hipóteses).
- **`checklists/`** — listas copiáveis por tema (antes/durante/depois).
- **`expanded/`** — aprofundamento sob demanda; um por CORE (01–12). Abra só quando o CORE não basta.
