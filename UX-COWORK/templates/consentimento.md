# Template — Consentimento informado

> Termo que o participante entende e aceita **antes** de qualquer registro. Usar em toda sessão gravada/com dados · NÃO usar como assinatura no automático (consentimento ≠ assinatura). Dono → CORE `11`.

## Como usar

Apresente e obtenha o aceite **antes** de qualquer nota ou gravação. Linguagem simples e recusa real possível. Reconfirme se o escopo mudar; menores exigem responsável + assentimento. Erro comum: gravar antes de explicar, ou usar trecho fora do combinado. **Este modelo não substitui jurídico — adeque à LGPD do seu contexto.**

---

```
CONSENTIMENTO INFORMADO

Objetivo do estudo: [ ___ ]
O que você fará: [ tarefas/conversa ___ ]   Duração: [ ___ ]

Gravação: [ ] tela  [ ] áudio  [ ] vídeo  — autoriza? [ ] sim  [ ] não
Dados coletados: [ ___ ]
Como os dados serão usados: [ ___ ]
Quem terá acesso: [ ___ ]
Anonimização: [ como os dados serão despersonalizados ___ ]
Retenção (por quanto tempo guardamos / quando apagamos): [ ___ ]

Observadores na sessão: [ ] sim  [ ] não  — autoriza? [ ] sim  [ ] não
Uso de trechos/citações (anonimizados): autoriza? [ ] sim  [ ] não
Incentivo: [ ___ ]

Você pode desistir a qualquer momento, sem prejuízo. [ ] entendi
Contato do responsável: [ ___ ]

Participante: [ nome ___ ]   Data: [ ___ ]   Aceite: [ ] sim
```

→ Entenda o método: CORE `11` (consentimento, LGPD, privacidade). Registro mínimo em contexto restrito → `14`.
