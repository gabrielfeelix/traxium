# Template — Intake de pesquisa

> Entrada de demanda de pesquisa. Usar **antes** de aceitar qualquer estudo · NÃO usar como balcão (nem todo pedido vira pesquisa). Dono → CORE `11`.

## Como usar

Puxe este formulário quando alguém pede uma pesquisa. A pergunta que decide tudo: **"que decisão será tomada com isto?"** — se ninguém responde, a demanda não está pronta, devolva. Erro comum: virar máquina de fazer estudo sem decisão por trás.

---

```
INTAKE DE PESQUISA

Solicitante: [ nome / área ___ ]
Produto / squad / fluxo: [ ___ ]
Contexto (o que está acontecendo): [ ___ ]

Decisão que esta pesquisa apoia: [ a decisão concreta que muda com o resultado ___ ]
Pergunta de pesquisa (o que precisamos aprender): [ ___ ]
Hipóteses (o que acreditamos hoje): [ ___ ]

Risco a reduzir: [ ] valor  [ ] usabilidade  [ ] viabilidade  [ ] negócio  [ ] adoção  [ ] navegação
Público a estudar: [ perfil ___ ]
Prazo (até quando a decisão precisa ser tomada): [ ___ ]
Evidência que já existe (analytics, suporte, pesquisa anterior): [ ___ ]
Dono da decisão (quem decide com base nisto): [ ___ ]

Impacto esperado (o que muda se aprendermos isto): [ ___ ]
```

→ Entenda o método: CORE `11` (lente de risco → `02` / `_METHOD_PICKER`).
