# Template — Relatório de usabilidade

> Readout enxuto de um teste. Usar para comunicar achados a quem decide · NÃO usar como relatório gigante que ninguém lê (proporcional ao risco). Dono → CORE `07`.

## Como usar

Preencha logo após analisar as sessões, enquanto fresco. Poucos achados fortes (3–7), por **severidade**, cada um ligado a evidência. Diga **quem os dados representam** e as limitações — não transforme 5 sessões em "60%". Erro comum: relatório longo sem decisão; se o stakeholder não sabe o que fazer ao terminar de ler, falhou.

```
RELATÓRIO DE USABILIDADE

Contexto (o que foi testado, em que estágio): [ ___ ]

Tarefas e cenários: [ ___ ]
Critério de sucesso (definido antes): [ ___ ]

Principais achados (3–7): [ ___ ]

Problemas por severidade:
  Crítico: [ ___ ]
  Alto:    [ ___ ]
  Médio:   [ ___ ]
  Baixo:   [ ___ ]

Evidências (fala/observação/clip/métrica por achado): [ ___ ]
Recomendações priorizadas (o que fazer, por quê, dono): [ ___ ]
Próximos passos (correção / reteste / nova pesquisa): [ ___ ]
```

→ Entenda o método: CORE `07` (severidade, condução). Como registrar o readout → `15`. Amostra/representatividade → `08`.
