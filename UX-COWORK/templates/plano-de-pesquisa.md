# Template — Plano de pesquisa

> Documento-base que alinha todos antes de coletar. Usar **depois** do intake aprovado, antes da 1ª sessão · NÃO usar para estudo sem decisão-alvo. Dono → CORE `01` / `11`.

## Como usar

Preencha quando o estudo foi priorizado e você vai executar. Serve para alinhar método, participantes e entregáveis antes de começar — não no meio. Erro comum: pular direto pro roteiro sem fixar a decisão-alvo e o critério de sucesso, e o estudo perder o rumo.

---

```
PLANO DE PESQUISA

Objetivo (o que queremos aprender): [ ___ ]
Decisão-alvo (que decisão isto apoia): [ ___ ]
Perguntas de pesquisa: [ 1. ___  2. ___  3. ___ ]
Hipóteses (o que pode falhar / esperamos): [ ___ ]

Método: [ entrevista / teste / card sort / tree test / survey / analytics… ___ ]
Por que este método (qual risco/dúvida ele reduz): [ ___ ]

Participantes (perfil + quantidade): [ ___ ]
Recrutamento (canal + critério): [ ___ ]  → usar screener
Roteiro de alto nível (blocos, não perguntas): [ ___ ]

Cronograma (etapas + datas): [ ___ ]
Entregáveis (o que sai disto): [ mapa de achados / relatório / recomendações ___ ]
Riscos e limitações (o que NÃO dá para concluir): [ ___ ]
Consentimento: [ como será coletado ___ ]  → usar consentimento
```

→ Entenda o método: CORE `01` (escolha de método) e `11` (operação). Registro do resultado → `15`.
