# Template — Log de hipóteses

> Registro vivo do que foi apostado, testado e decidido. Usar a cada hipótese levantada/testada · NÃO usar só para o que deu certo (o que falhou economiza mais retrabalho). Dono → CORE `15`.

## Como usar

Uma linha por hipótese, atualizada quando o resultado chega. Registre **o que falhou** (para não repetir) tanto quanto o que deu certo (para reusar) — é o ativo que mais se perde. Formato fixo do CORE `15`: hipótese → previsão → como testei → resultado → decisão. Erro comum: testar e não anotar o desfecho, e a próxima decisão voltar a ser por gosto.

Por linha: **Resultado** = confirmada / refutada / inconclusiva. **Decisão** = o que muda no produto a partir disto (inclusive "não fazer").

→ Entenda o método: CORE `15` (formato de registro). Formular a hipótese → `06`. Sem pesquisa formal, use proxy → `14`.
